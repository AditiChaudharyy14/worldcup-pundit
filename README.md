# WC Pundit

**AI football pundit + Hi-Lo streak game, live in Discord, trilingual (English / Nepali / Hindi), built on TxLINE World Cup 2026 data.**

- Live: **https://worldcup-pundit.onrender.com** (status page + `/health`)
- Join the Discord: see the link on the status page above
- Built for the TxODDS World Cup Hackathon (Consumer & Fan Experiences track)

## What it is

WC Pundit watches live TxLINE fixtures/scores/odds for World Cup 2026, turns
raw feed noise into clean fan-facing moments (goals, red cards, kickoff,
half-time, full-time, meaningful odds swings), and reacts to them like a
sharp, witty pundit reacting live -- in **English, Nepali, or Hindi**,
whichever the reader picked. Alongside the commentary, a Hi-Lo streak
mini-game asks "more or fewer than N combined goals this half?" at kickoff
and half-time, and tracks who's on the best guessing streak.

### Why trilingual, and why Discord

The original plan was Telegram, but Telegram is blocked in Nepal --
Discord isn't, and Nepali fans following the World Cup are exactly who a
Nepali-language pundit is for. Hindi covers a much larger neighboring
audience following the same tournament. English stays the default. Nobody
following a fixture yet still gets commentary -- it defaults to showing all
three languages rather than staying silent.

## Features

- **Live pundit commentary** -- one short (1-3 sentence) reaction per
  batch of events, generated by Claude, with a template-only fallback if
  the API call ever fails (the pipeline never stalls on a bad LLM call).
- **`/lang`** (or `/start`) -- pick English / Nepali / Hindi any time.
- **`/matches`** -- browse today's fixtures, follow/unfollow via buttons.
  Commentary for a fixture is only generated in the languages its
  followers actually use.
- **Hi-Lo streak game** -- automatic, no command needed: a "more or fewer
  than 2 combined goals this half?" question with two buttons posts at
  kickoff and again at half-time, resolves against the real score at the
  next boundary, and updates a per-player streak.
- **`/leaderboard`** -- top Hi-Lo streaks.
- **A public status page** (`/` and `/health`) -- bot uptime, events
  processed, last 5 events, the Hi-Lo leaderboard, and a "Join the Discord"
  button. This is the judges' entry point.

## Architecture

```
TxLINE devnet (SSE)                 Anthropic API
  /scores/stream  ---\                    ^
  /odds/stream    ----+--> Detector       |  one call per (fixture, language)
  /fixtures/snapshot -/    (Event         |  actually being followed
                           detection) --> Pundit / Dispatcher --> Discord
                              |                                  (embeds +
                              v                                   Hi-Lo
                          SQLite (dev)                            buttons)
                          or Postgres
                          (deployed)             FastAPI status page
                                                  (reads the same DB)
```

1. **`txline_client.py`** streams `/scores/stream` and `/odds/stream`
   (Server-Sent Events, with reconnect + backoff), and fetches
   `/fixtures/snapshot` for today's matches.
2. **`detector.py`** turns raw score/odds updates into clean `Event`s
   (`GOAL`, `RED_CARD`, `MATCH_START`, `HALF_TIME`, `MATCH_END`,
   `ODDS_SWING`), de-duplicated against a persistent idempotency ledger so
   replaying or reconnecting never double-fires an event.
3. **`pundit.py`** batches a fixture's events into one Anthropic call per
   language, using a few-shot persona prompt (never gambling advice --
   odds are explained as plain market-probability facts).
4. **`dispatcher.py`** fans that out per actively-followed language and
   posts rich embeds to Discord; **`game.py`** runs the Hi-Lo game the
   same way, off the same event stream.
5. **`web.py`** (FastAPI) serves the public status page and `/health` from
   the same database the bot writes to.
6. **`main.py serve`** runs all of the above -- bot, ingestion, and the web
   server -- as one process, which is what's actually deployed.

See **`docs/TECHNICAL.md`** for the event-detection details, the
deployment stack, and engineering findings (a couple of real bugs found
and fixed along the way). See **`docs/DEPLOY.md`** for the exact deployment
steps.

### TxLINE endpoints used

| Endpoint | Purpose |
|---|---|
| `GET /fixtures/snapshot` | Today's fixtures (teams, kickoff time) |
| `GET /scores/historical/{fixtureId}` | Full score history for replay (SSE-formatted despite the docs saying JSON) |
| `GET /scores/stream` | Live score updates (SSE) |
| `GET /odds/stream` | Live odds updates (SSE) |
| `POST /auth/guest/start` | Guest JWT (refreshed automatically on 401) |
| `POST /token/activate` | One-time API token activation (onboarding only -- see `docs/TECHNICAL.md`) |

## Setup (local dev)

Requires `onboarding/credentials.json` to exist (see `onboarding/subscribe.ts`
-- one-time on-chain subscribe + activation), Python 3.11+, and a repo-root
`.env` copied from `.env.example`.

```
python -m venv .venv
.venv/Scripts/pip install -r requirements.txt   # .venv/bin/pip on macOS/Linux
```

Fill in `.env` (see `.env.example` for every variable and what it's for):
at minimum `ANTHROPIC_API_KEY` for LLM commentary, and `DISCORD_BOT_TOKEN` +
`DISCORD_CHANNEL_ID` to run the Discord bot. `DATABASE_URL`/`DATA_DIR`/
`TXLINE_CREDENTIALS_JSON` are deployment-only (see `docs/DEPLOY.md`).

```
python -m app.main live --lang en                    # print live commentary to stdout
python -m app.main replay --fixture X --speed 10      # replay a finished fixture
python -m app.bot                                     # run the Discord bot
python -m app.main serve                              # bot + ingestion + status page, one process
```

Tests: `python -m pytest app/tests -v` (the end-to-end test hits the real
devnet API and is skipped automatically if `credentials.json` is absent).
